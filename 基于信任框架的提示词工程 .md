## 一、先说个大实话：为什么你的AI总像个"傻瓜"

![4b866eb9c70cb47a722b75b79a14cf23](https://img2024.cnblogs.com/blog/3821511/202607/3821511-20260705132519460-299630004.jpg)


### 从"人机交互简史"说起

| 时代       | 你在和谁说话       | 感觉怎么样                         |
| ---------- | ------------------ | ---------------------------------- |
| 日常生活   | 和人说话           | 累，太复杂了                       |
| 互联网时代 | 和电脑说话         | 钉是钉铆是铆，程序不会骗你         |
| **现在**   | **和AI智能体说话** | **……这玩意儿到底听不听得懂人话？** |

以前跟电脑打交道很简单——程序不会错，错了就是代码写错了。朋友会离开你，兄弟会背叛你，但**程序永远诚实**。

现在呢？AI横空出世，我们突然需要学会一种全新的沟通方式。于是网上冒出了无数教程：

👉 [Prompt Engineering Guide 提示工程指南](https://www.promptingguide.ai/zh)
👉 [智谱AI提示工程指南](https://docs.bigmodel.cn/cn/guide/start/introduction)
👉 [Spring AI 提示工程指南](https://docs.spring.io/spring-ai/reference/api/prompt.html#_prompt_engineering)

看完这些"奇淫巧计"，我的感受只有一个字：

### **乱。**

各种技巧满天飞：思维链、少样本学习、角色扮演、结构化输出、Re2重读法……背都背不完，更别说用了。

> **真正优秀的设计往往是简洁的。那些追求表面炫酷的东西，很快就会被淘汰。**

所以我在想：**如果用一个词把所有这些技巧串起来，会是什么词？**

## 二、我的答案：信任 🔗

![4b866eb9c70cb47a722b75b79a14cf23](https://img2024.cnblogs.com/blog/3821511/202607/3821511-20260705132519460-299630004.jpg)


没错，就是**信任**。

别急着划走，我不是要跟你聊什么心灵鸡汤。让我用一个最简单的比喻来说明这件事：

### 🧠 把AI想象成一个"全知全能的天才实习生"

这个实习生：

- 知识储备**极其丰富**（上知天文下知地理）
- 但有时候会**犯低级错误**（比如除法算错）
- 有时候会**一本正经地胡说八道**（AI幻觉）
- 不同版本的"实习生"能力**参差不齐**（模型差异巨大）

**你要做的不是找一个万能模板，而是学会判断：眼前这个实习生，我现在能信任他到什么程度？**

这就是本文的核心框架：

```
┌─────────────────────────────────────────────┐
│                                             │
│   ❌ 低信任 ────────────► ✅ 高信任         │
│                                             │
│   "我不确定你能做好"        "我相信你能搞定"  │
│   → 给详细指令              → 给方向就行     │
│   → 给示例模仿              → 让你自由发挥   │
│   → 一步步检查              → 看最终结果      │
│                                             │
└─────────────────────────────────────────────┘
```

### 什么叫会用AI,"好AI"？

AI的表现好坏，**只有两个原因**，而且它们没有高低之分，是相互制衡的：

| 原因                       | 通俗解释                       | 你能做什么                  |
| -------------------------- | ------------------------------ | --------------------------- |
| **原因一：AI能力不够**     | 这个任务对当前模型来说太难了   | 换更强的模型 / 降低任务难度 |
| **原因二：你的Prompt不行** | 你没把需求说清楚，或者方式不对 | 用更好的提示词技巧          |

**大多数人只盯着原因二疯狂学技巧，却忽略了原因一才是很多时候真正的元凶。**

## 三、我的三个真实踩坑故事 🩸

### 故事一：同一个Prompt，换个模型就活了

今年6月，我想让AI实现一个"记忆会话功能"——就是让AI记住之前聊过什么，不要每次都失忆。

我用DeepSeek试了，不行。
我用千问早期版本试了，也不行。
哪怕我把场景描述得再清楚，它还是记不住我是谁。

**然后我换了个千问3.7Max，一模一样的Prompt，一次就成功了。**

> 💡 **启示**：有时候不是你的Prompt写得烂，是这个AI真的做不到。

### 故事二：我和豆包吵了两个小时

两年前在学校做算法题，我花了一个小时算完答案，想找AI验证一下。结果豆包说我错了。

我和它**争论了整整两个小时**。

最后我去官网查了一下——**我的答案是对的，豆包做除法算错了**。

而且最气人的是：**它打死不认错**。AI幻觉这个东西，真的会让你怀疑人生。

> 💡 **启示**：AI也会犯低级错误，而且犯错的时候特别自信。**永远不要100%相信AI给出的答案。**
> ![855b7a95ee5da2b6bb3c0d71fb8f50e0](https://img2024.cnblogs.com/blog/3821511/202607/3821511-20260710174037860-1611033212.png)

### 故事三：三个AI，一道题，三种翻车现

有一次我看到一道经典的逻辑题（来源：[pdai.tech算法概览](https://www.pdai.tech/md/algorithm/alg-basic-overview.html)），自己想了很久没想出来，于是同时丢给了Kimi、DeepSeek和豆包：

> **题目**：12个球和一个天平，知道只有1个球重量不同（不知道偏轻还是偏重），怎么用**三次**称出那个球？

结果呢？三个AI表现各不相同：

- 有的推理过程看起来头头是道，结论却是错的
- 有的直接给出了一个根本无法操作的方案
- 有的中途就开始自相矛盾

> 💡 **启示**：复杂的逻辑推理任务，即使是目前最强的AI也可能翻车。这种情况下，你需要的是**更低信任度的提示策略**（后面会讲）。

## 四、那什么时候该用"低信任"，什么时候该"高信任"？🎚️

### 先搞清楚一件事

**信任不要放在AI的能力上**——因为AI的能力必然会越来越强，这是大趋势。

**信任应该放在"当前这次协作"上**——也就是：

- 这个AI模型 + 这类任务 + 我的Prompt = 能不能达成目标？

当你发现AI的表现不尽人意时，问自己一个问题：

> **从低信任到高信任，中间缺的是什么？**

- 缺的是**更好的Prompt**？→ 原因二，往下看技巧
- 缺的是**更强的模型**？→ 原因一，考虑换工具

### 一个形象的比喻

用5090显卡带诺基亚手机去打王者荣耀——

- 🎮 **5090显卡**：超强性能，应该去跑实验室项目或3A大作
- 📱 **诺基亚手机**：经典好用，但不是干这个的
- ⚔️ **王者荣耀**：好游戏，但别在这种荒诞组合下打

三者我都喜欢，硬凑在一起就是浪费。

> **好的Prompt = 让对的AI，用对的方式，做对的事。**

<a id="section-low-trust"></a>

### 🔒 低信任模式：当你还不太确定AI能不能做好时

**适用场景**：

- 第一次尝试某类任务
- 任务复杂度高，容错率低
- 使用的模型能力未知或较弱
- 对输出准确性要求很严格

**核心思路**：把AI当成一个需要手把手教的实习生，**多说、多给例子、多检查**。

---

#### 技巧1：明确指定角色 + 具体任务

> **原理**：告诉AI"你是谁"和"你要干什么"，比直接扔一个问题效果好10倍。

📋 **复制测试**：

```
系统：你是一位经验丰富的Python教师，擅长向初学者解释编程概念。请使用通俗易懂的语言，适当使用生活中的类比来帮助理解。

用户：请解释 Python 中的列表推导式（List Comprehension），包括基本语法和 2-3 个实用示例。
```

---

#### 技巧2：提供详细说明 + 输出格式模板

> **原理**：AI最怕"模糊"。你越具体，它的输出就越可控。

📋 **复制测试**：

```
请为一个新上市的智能手表制定社交媒体营销计划。

计划应包含以下内容：
1. 目标受众描述（年龄、职业、兴趣特征）
2. 三个内容主题（每个主题用一句话概括）
3. 每个平台的内容类型建议（至少覆盖小红书、抖音、微博）
4. 发布频率建议

输出格式要求：
━━━━━━━━━━━━━━━━━━━━━
📌 目标受众：[具体描述]
📌 内容主题：[主题1] / [主题2] / [主题3]
📌 平台策略：
   • [平台名] — [内容类型] — [发布频率]
   • [平台名] — [内容类型] — [发布频率]
━━━━━━━━━━━━━━━━━━━━━
```

---

#### 技巧3：用表格/列表等结构化格式引导思维

> **原理**：结构化的输入会带来结构化的输出。散乱的文字只会换来散乱的回答。

📋 **复制测试**：

```
分析特斯拉（Tesla）公司的竞争优势和潜在劣势。

请严格按照以下表格格式回答：

| 维度 | 具体内容 | 简要分析 |
|------|---------|---------|
| 优势1 | [填写] | [一句话解释为什么是优势] |
| 优势2 | [填写] | [一句话解释为什么是优势] |
| 优势3 | [填写] | [一句话解释为什么是优势] |
| 劣势1 | [填写] | [一句话解释为什么是劣势] |
| 劣势2 | [填写] | [一句话解释为什么是劣势] |
| 劣势3 | [填写] | [一句话解释为什么是劣势] |
| 应对建议 | [填写] | [针对劣势的改进方向] |
```

---

#### 技巧4：明确约束输出格式（字数、风格、结构）

> **原理**：不给限制，AI就会自由发挥；而自由发挥的结果往往不是你想要的。

📋 **复制测试**：

```
请撰写一篇关于气候变化的科普文章。

【严格遵循以下要求】
✅ 目标读者：高中生水平
✅ 文章结构：5个小标题，每个标题下2-3段文字
✅ 总字数：800字左右（不超过900字）
✅ 语言风格：通俗易懂，避免专业术语堆砌
✅ 结尾必须包含：3条普通人可以立即执行的个人行动建议

⛔ 不要做的事：
- 不要使用"碳中和""温室效应"等专业词汇而不加解释
- 不要写成危言耸听的末日论调
- 不要超过字数限制
```

---

#### 技巧5：思维链（Chain-of-Thought）——让AI展示思考过程

> **原理**：对于数学、逻辑推理类问题，让AI"把过程写出来"比直接给答案准确率大幅提升。

📋 **复制测试**：

```
【数学应用题】
一个商店售卖T恤，每件15元。如果购买5件以上可以享受8折优惠。小明买了7件T恤，他需要支付多少钱？

【答题要求】
请不要直接给出最终答案。请按照以下步骤逐步写出完整的思考过程：

第1步：计算7件T恤的原价（列出算式）
第2步：判断是否满足折扣条件（说明理由）
第3步：计算折扣后的单价（列出算式）
第4步：计算最终支付金额（列出算式）

每一步都要写出数字和单位，确保过程清晰可查。
```

---

#### 技巧6：少样本学习（Few-Shot）——给AI几个"标准答案"照着抄

> **原理**：人类学习最快的方式是模仿。AI也一样。给它几个完美的范例，它就会照着这个模式输出。

📋 **复制测试**：

```
我要你做情感分析任务。我会给你一些用户评论，请你判断每条评论的情感倾向（正面/负面/中性），并简要说明理由。

以下是几个示例（请严格按照此格式输出）：

【示例1】
输入："这家餐厅的服务太差了，等了一个小时才上菜"
输出：❌ 负面 | 理由：抱怨等待时间长和服务质量差

【示例2】
输入："新买的手机屏幕清晰，电池也很耐用"
输出：✅ 正面 | 理由：赞扬了屏幕和电池两个产品优点

【示例3】
输入："物流速度还行，但是包装有点简陋"
输出：⚠️ 中性 | 理由：既有肯定也有批评，整体态度平和

━━━━━━━━━━━━━━━━━━━━━
现在请分析这条评论：
"这本书内容还行，但是价格有点贵"
请按照上面的格式给出你的分析：
```

#### 技巧7：分步骤指导（Step-by-Step）——复杂任务拆解

> **原理**：一次性给AI一个大任务，它容易遗漏细节。拆成小步骤，一步步来，效果更好。

📋 **复制测试**：

```
我需要你帮我设计一个网站落地页方案。请严格按照以下5个步骤依次完成，每一步完成后再进行下一步：

【步骤1】目标受众分析
- 考虑用户的年龄、职业、核心需求和痛点

【步骤2】核心信息提炼
- 确定主标题（1句话）、副标题（2-3句话）、价值主张

【步骤3】页面结构规划
- 列出页面应包含的所有区块（至少5个区块）

【步骤4】视觉引导策略
- 推荐配色方案、关键视觉元素建议

【步骤5】行动召唤设计
- 设计CTA按钮文案和放置位置

请开始步骤1：
```

---

#### 技巧8：自我评估与修正 —— 让AI自己检查自己的答案

> **原理**：AI生成内容后，让它以"第三方评审"的身份重新审视一遍，能大幅降低错误率。

📋 **复制测试**：

```
【第一轮】请解决以下概率问题：
从一副标准扑克牌（52张）中随机抽取两张牌，求抽到至少一张红桃的概率。

【第二轮 - 自检环节】
在你给出答案之后，请立刻以"严格的数学老师"身份，对你自己的解答进行逐项审查：

□ 你的推理过程是否存在逻辑漏洞？
□ 你使用的概率公式是否正确？
□ 计算步骤是否有算术错误？
□ 是否有遗漏的情况没有考虑到？

如果发现任何问题，请给出修正后的完整解答。如果没有问题，请说明"经检验，解答正确"并给出置信度评分（1-10分）。
```

---

#### 技巧9：设置明确的边界约束 —— 告诉AI"不要做什么"

> **原理**：有时候告诉AI"不能做什么"比告诉它能做什么更重要。这能有效避免AI跑偏或产生幻觉。

📋 **复制测试**：

```
请用不超过200字介绍Spring框架的核心思想。

【严格遵守以下约束】
❌ 不要列举具体的模块名称（如Spring MVC, Spring Boot, Spring Cloud）
❌ 不要出现任何代码片段
❌ 不要使用过于技术化的术语（如"依赖注入""控制反转"除非你用通俗语言解释）
❌ 不要写成广告宣传口吻
✅ 要做到：让一个完全不懂编程的人也能看懂Spring解决的是什么问题
```

---

#### 技巧10：迭代式优化 —— 从粗糙到精细

> **原理**：好的Prompt很少一次成型。根据AI的反馈不断调整，就像调试代码一样。

📋 **复制测试（三轮迭代示范）**：

```
【第1轮 - 初版】
谈谈人工智能的影响。
→ （预期：AI会给一个泛泛而谈的回答）

【第2轮 - 收窄范围】
分析人工智能对医疗行业的三大积极影响和两大潜在风险，每个影响/风险都需要提供具体的应用案例。
→ （预期：AI会更聚焦，但仍可能不够深入）

【第3轮 - 精确到场景】
详细分析AI在医学影像诊断领域的具体应用：
1. 列举2-3个已经商业化落地的AI诊断系统及其公开报道的准确率数据
2. 说明这些系统如何辅助放射科医生的工作流程
3. 分析实施过程中面临的最大挑战（技术层面+伦理层面）
4. 预测未来3-5年可能的技术突破方向
→ （预期：AI会给出非常具体、有数据支撑的回答）
```

### 🔓 高信任模式：当AI已经证明自己靠谱时

**适用场景**：

- 使用的是较强的模型（GPT-4/Claude 3.5/GPT-4o等）
- 已经通过多轮对话建立了上下文默契
- 任务需要创意和发散性思维
- 追求效率大于追求精确

**核心思路**：把AI当成一个靠谱的合作伙伴，**给方向、给目标，剩下的让它自由发挥**。

---

#### 技巧11：赋予角色 + 开放式目标

> **原理**：高信任模式下不需要手把手教。定义清楚"你是谁"和"我们要去哪里"，让AI自己选路。

📋 **复制测试**：

```
【你的角色】
你是一位拥有10年实战经验的产品增长负责人，擅长通过数据驱动和用户洞察来制定增长策略。你既理性又富有创造力，善于在数据中发现机会。

【我们的挑战】
我们的在线教育平台用户留存率在注册后第7天出现明显断崖——从首日60%骤降至25%。这意味着每4个注册用户中有将近3个在一周内流失。

【你的任务】
1. 分析导致这一断崖现象的最可能的三类原因
2. 为每类原因设计一个可落地的A/B测试实验方案

【探索方向参考】
可以从用户激活体验、价值感知时机、社交关系链等维度切入，但不限于此。

【产出要求】
每类原因需要包含：原因分析（2-3句话）+ 实验方案（假设、变量、预期指标、最小样本量估算）
```

---

#### 技巧12：开放式头脑风暴

> **原理**：有些任务没有标准答案，就是要天马行空。这时候约束太多反而限制了AI的创造力。

📋 **复制测试**：

```
【话题】可持续时尚（Sustainable Fashion）

【问题】
在未来5-10年，"可持续时尚"领域可能会出现哪些具有颠覆性的技术创新或商业模式变革？

【要求】
- 请展开想象，不受现有技术限制
- 列出你认为最具潜力的5个方向
- 每个方向用3-5句话阐述：是什么 + 为什么有颠覆性 + 目前的主要障碍
- 不需要固定的输出格式，期待你有独特甚至大胆的见解
- 如果某个想法听起来"疯狂"，请不要自我 censor，大胆写出来
```

---

#### 技巧13：基于上下文的深度追问

> **原理**：当你们已经聊了很久、建立了共同语境之后，不需要每次都重复背景信息。简洁地追问即可。

📋 **复制测试**：

```
（以下是在我们已经深入讨论了某个分布式系统的架构方案之后的追问）

回到刚才讨论的"数据一致性"挑战——

如果我们采用你提到的第二种方案（最终一致性 + 事件驱动架构），那么在分布式事务的具体实现上：

1. 你会推荐哪个开源框架或模式？（如Saga、TCC、本地消息表等）
2. 选择这个方案的核心考量是什么？
3. 在我们的业务场景下（日均百万订单），可能的性能瓶颈在哪里？
```

---

#### 技巧14：让AI当"严厉的评审官"

> **原理**：高信任模式的AI有能力进行高质量的自我反思。让它站在对立面审视自己的工作，往往能发现盲点。

📋 **复制测试**：

```
【背景】
这是我们团队刚刚起草的一份产品迭代计划（见上方内容）。

【你的任务】
请你暂时忘掉你是这份计划的参与者。现在你是一位来自顶级咨询公司的**首席评审官**，以最严苛的标准审视这份计划。

请指出：
1. 🔴 最大的3个风险点或致命缺陷（如果有）
2. 🟡 3个被忽视或低估的潜在问题
3. 🟢 3个值得肯定的亮点

对于每个问题和风险，请给出：
- 具体是哪里有问题
- 可能导致的后果
- 你的修改建议

不要客气，该批评就批评。越严厉越好。
```

---

#### 技巧15：多视角全方位分析

> **原理**：复杂问题往往没有唯一正确的角度。让AI同时戴上不同的"眼镜"来看同一个问题，能得到更全面的认知。

📋 **复制测试**：

```
【议题】某城市正在讨论一项提案：禁止私家车进入市中心区域。

请你从以下4个截然不同的视角，分别进行分析：

🌿 视角1：环保专家
- 支持：2个最强论点
- 反对：2个最大顾虑
- 折中方案建议

💰 视角2：经济学家
- 支持：2个最强论点
- 反对：2个最大顾虑
- 折中方案建议

🏪 视角3：市中心商户代表
- 支持：2个最强论点
- 反对：2个最大顾虑
- 折中方案建议

🚶 视角4：每天开车通勤的普通市民
- 支持：2个最强论点
- 反对：2个最大顾虑
- 折中方案建议

【最后】
综合以上4个视角，给出你的个人立场和理由。
```

---

#### 技巧16：知识检索 + 来源标注

> **原理**：对于需要事实支撑的任务，要求AI标注信息来源，既能提高可靠性，也方便你事后核查。

📋 **复制测试**：

```
请解释光合作用的过程及其在生态系统中的作用。

【回答要求】

1️⃣ 科学定义：用一段话准确定义光合作用

2️⃣ 化学反应：写出主要的化学反应方程式（文字形式即可）

3️⃣ 关键因素：列出影响光合作用效率的5个最重要因素

4️⃣ 生态意义：说明光合作用如何支撑整个地球生态系统的能量流动

⚠️ 特别注意：
- 对于涉及具体数据的陈述（如"光合作用转化了约XX%的太阳光能"），请标注信息来源及可靠性评估
- 如果你引用了某项研究或实验，请注明研究者/机构和大致年份
- 对于不确定的信息，请明确标注"此数据可能存在争议"
```

## 六、🎚️ 动态校准：你的"信任度仪表盘"

"低信任"和"高信任"不是非黑即白的，它们是一个**连续的光谱**。你需要根据AI的实际反馈，**动态调整**你的策略。

### 校准方法（3步走）

```
第1步：默认从"中等偏低信任"开始
       ↓
       使用清晰的指令 + 基础格式化要求
       ↓
第2步：观察AI的反馈信号
       ↓
    ┌─────────────────────────────────────┐
    │  ✅ 正向信号（可以考虑提升信任度）   │
    │  · 回答精准到位                     │
    │  · 严格符合你要求的格式             │
    │  · 逻辑清晰有见地                   │
    │  · 能主动关联之前的上下文           │
    ├─────────────────────────────────────┤
    │  ❌ 负向信号（需要降低信任度）       │
    │  · 答非所问，理解偏差大             │
    │  · 忽略了你设置的格式要求           │
    │  · 出现明显的事实性错误             │
    │  · 推理过程混乱或跳跃               │
    └─────────────────────────────────────┘
       ↓
第3步：采取行动
       ↓
    正向信号多 → 下轮尝试更高信任度（减少指令、增加开放度）
    负向信号多 → 下轮降低信任度（增加约束、提供更多示例）
```

### 一句话记住心法

> **你的每一次提问，都是在调节你和AI之间的"信任旋钮"。**

旋钮调得太高（过度信任），AI会放飞自我；
旋钮调得太低（不信任），你又累又效率低。

**找到合适的刻度，这才是Prompt工程的终极奥义。**

## 七、写在最后：从"下命令"到"建关系"

市面上99%的Prompt教程都在教你一件事：**怎么给AI下命令**。

但这篇文章想告诉你的是另一件事：**怎么和AI建立一种高效的协作关系**。

|          | 弱智能时代（现在）                 | 强智能时代（未来）                   |
| -------- | ---------------------------------- | ------------------------------------ |
| 你的策略 | 低信任为主：多引导、多约束、多检查 | 高信任为主：给方向、给目标、让AI发挥 |
| 核心目标 | **弥补AI的不足**，确保可靠         | **释放AI的潜能**，追求创造           |
| 你的角色 | 严谨的"导师"                       | 信任的"合作伙伴"                     |

### 什么是真正的Prompt高手？

![2e1e6d4b0946903903f685e6bb1700c8](https://img2024.cnblogs.com/blog/3821511/202607/3821511-20260705132449365-715109066.jpg)

不是背下了100个模板的人，
也不是掌握了最新最炫技巧的人，

而是**最懂得判断当下该给AI多少信任、并能精准地把这种信任度通过Prompt传达出去的人**。

这是一种**元能力**——它比任何单个Prompt技巧都更有价值，因为它永远不会过时。
